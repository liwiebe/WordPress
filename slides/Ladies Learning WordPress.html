What is WordPress?

An Open Source Content Management System used to power thousands of websites

Initially was used to power blogs

Powerful enough to run large websites and applications

Used by TechCrunch, CNN, Ladies Learning Code and many more...

WordPress.com vs. WordPress.org

WordPress.com is hosted for you - no servers or code required!

It doesn't allow for a lot of customization, though. No plugins or custom themes.

We will be using WordPress.org

This allows for 100% customization - themes and plugins oh my!

Code + Database = ♥

The database holds our data and the code prints it to the page. The two work together to output HTML

/wp-admin and /wp-includes holds the engine of WordPress - we don't touch it.

/wp-content holds our specific content like themes, plugins and image uploads.

What you need to have installed or downloaded:

An application that can run your local server (MAMP, WAMP or XAMPP)

A text editor like Sublime Text 2

Your WordPress ZIP file from WordPress.org

Let's discuss each of these a bit more:

Localhost

Local Server (MAMP, WAMP or XAMPP): A small service that runs on your laptop which mimics the functionality of a large, paid server.

We are able to work on our website without uploading it anywhere.

HTML review

HTML stands for "hyper-text markup language" and it's the main language your browser understands.

HTML works by "marking up" your text with tags that explain what that text is.

In Wordpress, PHP is used to control which HTML gets displayed. HTML and PHP live together in .php files. You'll see this in action shortly.

CSS review

CSS stands for "cascading style sheets" and it's a powerful way to control the look and feel of web pages.

CSS tells the browser how the content inside specific HTML tags should be displayed.

It's common to use an attribute like a class or ID to label parts of your HTML so that CSS can control it more directly.

CSS is often held in a separate .css file and linked to in the head area of a webpage.

In Wordpress, CSS is stored in a file inside your theme, often called style.css.

More on CSS a bit later in the class!

PHP

You've probably noticed that a lot of files in your WordPress folders end with ".php"

PHP is the programming language WordPress is written in. The server uses the PHP to decide what to show your visitors and put it all together.

PHP tags: HTML and PHP often live together. When we need to use PHP, we put it inside <?php and ?> tags, so our server will know to process it as PHP.

How Wordpress builds each page

Every page on your site has a layout that comes from your theme.

The layout gets filled with content from your database.

Your theme can have many different layouts, and each option lives in a template file made out of HTML and PHP.

Example: your home page (index.php) might look different than your category page (category.php)

Let's take a look at some of the more important template files in our theme:

index.php - The front page of your blog

single.php - Default template used for displaying a single post

category.php - Displays all posts within a certain category

footer.php - The footer that is appended at the bottom of every page

header.php - The header that is appended to the top of every page

sidebar.php - Sidebar that is appended to every page

loop.php - A reusable file to display posts (much more on this later!)

Anatomy of a WordPress Theme

A single page is often made up of multiple template files that work together to display all the information you need.

For Example...

single.php is used to show a single blog post.

Inside single.php, other template files are called in:

  • header.php contains the top section
  • footer.php contains the bottom section
  • sidebar.php contains the sidebar
  • loop.php is used to pull in posts.

Theme Layout Basics

Most template pages can be stripped down the following:

<?php get_header(); ?>
      { The Page's Content and the Loop live Here}
    <?php get_sidebar(); ?>
  <?php get_footer(); ?>

Introduction to The Loop

The loop is the lifeblood of a WordPress theme. It gets your posts and pages, and all the info that goes with them.

Why is it called "The Loop"? Because it does the same thing over and over, until it's told to stop.

Here's an example: on your blog page, it's used to display your most recent blog posts over and over - until it hits the limit that we set in our Settings.

We set it to 5 earlier. So our blog page loop shows 5 posts.

A single post (single.php) only shows 1 post. But it uses the same loop.

So what?

The Loop is what makes WordPress work

On every page, you have opening tags, title, content, closing tags. The code is the same every time.

It's the loop that pulls out and displays all the great content you set up in your Wordpress dashboard.

The loop keeps your content separate from the the design, or theme, of your site.

Let's learn more about the loop, so that we can change what types of content it pulls out.

How a Wordpress page gets put together

1. The blog page is requested by the user. Wordpress asks the database for all content for that page.

2. The database hands back the 5 most recent blog posts, in order.

3. Wordpress starts with header.php and starts to draw the page.

4. Wordpress then gets to the loop. Since we have 5 posts from the database, we run the loop 5 times.

5. Wordpress is done with the loop, so it moves on, and outputs the sidebar and the footer. Page complete!

The Code

A stripped down look outputting only the title and the content.

<?php if ( have_posts() ) while ( have_posts() ) : the_post(); ?>

          <h2><?php the_title(); ?></h2>
        <?php the_content(); ?>

      <?php endwhile; ?>

Still Confused? Don't Worry!

Our WordPress theme comes with the loop.php which handles most of this for you

You just need to be aware of how the loop works and what it looks like so you can edit it!

Template Tags

How does content get onto the page while we're in The Loop?

Wordpress provides template tags, short little PHP functions that output content from your site.

No knowledge of PHP is required.

Most template tags only work within the loop, but others are global and can be used anywhere on the site.

In fact, we've already used some of them!

CSS 101

We're going to start customizing the look of our WordPress theme next! But first, let's do a refresher on CSS. We won't be diving too deep, as it's a huge topic on its own, but let's take a quick look.

With CSS, you can easily add styling to the elements on your website. Examples of elements are heading tags, images, divs (content and sidebars), links and lists.

A basic CSS definition has three parts: selector, property and value.

CSS Selectors

The Selector defines which element the CSS will be applied to.

<div id="wrapper"> ... </div> 
    div#wrapper { ... }

  <div class="post"> ... </div> 
  div.post { ... }

<span class="date"> ... </span> 
span.date { ... }

<span class="category"></span> <div class="meta"></div>  
span.category, div.meta { … } 

<div class="parent"><div class="child"></div></div> 
div.parent div.child { ... }

CSS Properties and Values

The property defines which style is being set on the current selector and the value sets it.

There are hundreds of CSS properties. Here a few popular ones for a div with the ID of wrapper

#wrapper {
      background : black;
      border:2px solid green;
      font-family:Georgia, sans-serif; 
    }

Let's go through a few now...

a {
    text-decoration: none; /* Removes all underlines from links */
    color: blue; /* Changes all links to blue */
  }
a:hover {
    color: red; /* Changes all links to red  only when we hover*/
  }
.post {
    background : url(relative/pathp/to/the/background.png); /* Add an image to the background */
    border: 3px solid yellow; /* add a yellow border */
    padding:20px; /* Increase padding to 20px */
  }
.post h2 a { /* finds .post -> Heading 2 -> links */
    font-size:40px; 
    font-family: georgia, sans-serif; /* change the font */
    border-bottom:3px solid goldenRod; /* Changes the bottom border only to goldenRod colour*/
  }
ol li { /* Selects all ordered list items */
    list-style-type: upper-roman;  /* Changes from 1,2,3 to I, II, III...*/
  }
list-style-type: square; /* changes from circle bullets to squares*/
   font-style:italic;
 }
.more {
 float: left/right/none; /* floats an element to the left or right of its relative parent */
 text-align: center/left/right/inherit;
 margin: 10px 20px 15px 25px; /* Top Right Bottom Left margin values*/
 font-weight:bold;
 width:100%;

 overflow:scroll; /* anything over 500px high will have scrollbars */
}
